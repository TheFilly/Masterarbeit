# Documentation

`docs/` stores research findings, decisions, and thesis traceability. Keep raw
notes out of `PLAN.md`; promote only reviewed material into decisions.

## Directories

- `research/`: phase summaries, open questions, findings, evidence, and matrices.
- `decisions/`: accepted or proposed architecture and scope decisions.
- `thesis/`: claim register and chapter mapping.
- `templates/`: required formats for new findings, decisions, and claims.
- `archive/`: superseded material. Do not cite it as current evidence.

## Reading Order

1. Read the relevant phase `summary.md`.
2. Read `open-questions.md`.
3. Open only linked findings, evidence, or decisions.
4. If no current summary exists, note the gap before using lower-level files.

Accepted decisions outrank draft findings. If summaries lag behind findings,
update the summary instead of relying on unstated context.
